Как работает Test Loop

Это не маркетплейс кредитов. Это система взаимного тестирования: разработчики обмениваются тестами приложений, подтверждают качество через daily proof и строят репутацию.

Главный Flow обмена

От запроса до подтверждения - прозрачный пайплайн с понятными статусами на каждом шаге.

01

Request Exchange

User A выбирает приложение User B в маркете и отправляет заявку. Создается exchange со статусом pending.

02

Incoming Request

User B получает входящий invite в Home и решает: принять или отклонить.

03

Accept & Start

При принятии exchange становится active, выставляются startDate и endDate = +14 days.

04

Daily Execution

В течение 14 дней обе стороны проходят daily proof-ритуал: загружают подтверждения, проверяют партнера, закрывают дни.

Proof-система (ядро доверия)

Каждый день пользователь отправляет proof, а другая сторона подтверждает качество выполнения.

Flow отправки proof

  • Открыть приложение партнера
  • Сделать скриншоты
  • Загрузить до 3 фото
  • Отправить на проверку

Flow проверки

  • Партнер получает уведомление
  • Смотрит фото и день
  • Жмет Approve или Reject
  • При Reject пользователь пересдает

Анти-чит и репутация

Экономика продукта строится на trust-score: дисциплинированные пользователи получают больше обменов и выше приоритет.

Ограничения

  • Максимум 3 фото в proof
  • Нельзя отправить proof без фото
  • Нельзя спамить отправкой без ответа

Success path

Все 14 дней approved -> exchange = completed, увеличиваются completedExchanges и растет репутация.

Failure path

Пропуски и невыполнение обязательств -> exchange = failed, репутация снижается, доверие в системе падает.

FAQ

Короткие ответы на самые частые вопросы по процессу обменов.

Что происходит, если proof отклонен?

Статус proof становится rejected. Пользователь получает фидбек и пересдает proof за этот же день.

Можно ли отправить proof без фото?

Нет. В системе есть анти-чит правила: без фото отправка блокируется.

Когда exchange считается завершенным?

Когда все 14 дней подтверждены обеими сторонами: exchange = completed.

Что дает репутация?

Репутация влияет на доверие в системе и на вероятность получать новые обмены.
